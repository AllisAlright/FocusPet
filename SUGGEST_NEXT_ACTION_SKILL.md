# FocusPet Skill 设计文档：suggest_next_action（下一步建议）

## 1. Skill 定义

`suggest_next_action` 是 FocusPet 的第二个正式 AI Skill。

它的核心作用是：

> 当用户不知道先做什么、任务卡住、或页面停留无操作时，给出一个**明确、低摩擦、可立刻开始的下一步建议**。

这个 Skill 的目标不是做泛泛鼓励，也不是生成复杂计划，而是：

- 帮用户从“停滞”进入“行动”
- 降低决策成本
- 让 AI 在关键时刻给出一个刚刚好的推动

它和 `split_task` 的区别是：

- `split_task`：把“大目标”拆成几步
- `suggest_next_action`：在“现在这一刻”，告诉用户先做哪一步

---

## 2. 为什么要做这个 Skill

在 FocusPet 的使用场景里，用户经常会遇到以下情况：

- 打开 App，但不知道先做什么
- 待办很多，但没有开始动作
- 某个任务进行到一半停住了
- 明明有任务，却一直没有进入专注

这说明产品里不仅需要“拆任务”的能力，还需要一种更轻、更即时的能力：

> **在用户犹豫或停滞时，给出一个明确的下一步。**

`suggest_next_action` 正是为这个场景设计的。

---

## 3. Skill 目标

`suggest_next_action` 的目标是输出：

- 一条建议
- 一句话
- 明确、具体、容易开始
- 与当前任务状态相关
- 不增加用户负担

它强调的不是“分析”，而是：

> **替用户减少一点犹豫，让用户更容易开始。**

---

## 4. 输入与输出

## 4.1 输入

MVP 阶段建议输入如下：

- 当前未完成任务列表
- 每个任务的基本状态：
  - 标题
  - status
  - progress
  - spent_minutes
- 最近 focus session 信息（可选）
- 当前页面（可选）

为了保持简单，第一版可以只使用：

- `tasks: List[TaskSummary]`

其中每个任务可包含：

- `id`
- `title`
- `status`
- `progress`

后续再逐步扩展为完整上下文输入。

---

## 4.2 输出

输出为一条简短建议：

```json
{
  "message": "先把自我介绍草稿写出第一版。"
}
```

要求：

- 只有一句话
- 明确可执行
- 尽量低摩擦
- 适合作为“此刻就去做”的下一步

---

## 5. 输出原则

`suggest_next_action` 必须遵循以下规则：

### 5.1 只给一条建议
不要给多个选项，避免增加决策成本。

---

### 5.2 先选最容易开始的一步
优先推荐：

- 标题清晰
- progress 较低但可启动
- 不需要太多准备的任务

---

### 5.3 建议必须具体
例如：

- ✅ 先把自我介绍写成 3 句话
- ❌ 建议你开始准备面试

---

### 5.4 语气温和
建议不应命令式，也不应太工具化。

例如：

- ✅ 可以先把最容易开始的一步做掉
- ✅ 先写一个小草稿也很好
- ❌ 立即执行以下任务

---

### 5.5 与当前任务状态有关
如果当前任务列表为空，就不要假装给任务建议；应返回更基础的引导。

例如：

- 当前没有任务 → “先写下一件你最想推进的事吧。”

---

## 6. 当前适用场景

这个 Skill 最适合出现在以下场景：

### 6.1 首页轻建议卡
用户停留在首页时，给一句“现在可以先做什么”。

### 6.2 待办页顶部建议
用户看到一堆任务但没有动作时，给一个推荐下一步。

### 6.3 拆任务后的自然衔接
用户完成 `split_task` 并导入待办后，可以自动推荐：

- 先从哪一条开始
- 哪一条最适合现在推进

---

## 7. 与 `split_task` 的关系

`suggest_next_action` 不是替代 `split_task`，而是与它互补。

### `split_task`
解决的是：
- “目标太大，不知道怎么拆”

### `suggest_next_action`
解决的是：
- “我有任务了，但现在不知道先做什么”

两者一起构成 FocusPet 最基础的 AI 行动支持系统：

- 先拆开目标
- 再推动用户迈出第一步

---

## 8. MVP 实现建议

为了保持简单，第一版建议采用：

### 8.1 轻量结构化 Skill
和 `split_task` 一样，将其实现为正式 Skill：

- `name`
- `description`
- `run(...)`

例如：

- `name = "suggest_next_action"`
- `description = "Suggest one small, actionable next step based on the user’s current unfinished tasks."`

---

### 8.2 后端输入简化
第一版不做复杂上下文聚合，不做完整 Context Engine，只传：

- 当前未完成任务列表

后端自己做轻量逻辑：

- 如果有 `in_progress` 任务，优先从中选一条
- 如果没有，则从 `todo` 中选最容易开始的一条
- 再交给 LLM 生成一句更自然的建议

这样实现简单，而且已经能体现 Skill 的价值。

---

### 8.3 前端输出简单
第一版前端不需要复杂页面，只要有一个轻量展示入口即可。

例如：

- 首页一句 AI 建议
- 一个“换一个建议”按钮（可选）
- 点击后跳到对应任务（可选）

---

## 9. 当前后端实现建议

建议新增：

### 后端接口
- `POST /api/v1/ai/suggest-next-action`

请求示例：

```json
{
  "tasks": [
    {
      "id": "1",
      "title": "准备自我介绍",
      "status": "todo",
      "progress": 0.0
    },
    {
      "id": "2",
      "title": "整理项目案例",
      "status": "in_progress",
      "progress": 0.4
    }
  ]
}
```

响应示例：

```json
{
  "message": "先把正在推进的项目案例整理成 3 个要点。"
}
```

---

## 10. Prompt 策略建议

`suggest_next_action` 的 prompt 应区别于 `split_task`。

### 重点约束
- 只输出一句建议
- 基于当前任务状态
- 优先推荐最容易开始的一步
- 不泛泛而谈
- 不给多个选项
- 保持温和语气

### 推荐风格
像这样：

- 先把正在做的项目案例整理成 3 个要点。
- 先写出自我介绍的第一版也很好。
- 先把最容易完成的一步做掉，比如列出 3 个高频问题。

---

## 11. 与 Agent 架构的关系

`suggest_next_action` 在 FocusPet 的 Agent 架构中，对应的是第二个正式 Skill。

### Skills 层
它是独立 Skill，与 `split_task` 并列。

### Orchestrator 层
未来可根据用户意图调用：

- 用户说“不知道先做什么” → `suggest_next_action`
- 用户说“帮我拆一下任务” → `split_task`

### Context Engine
未来可引入：
- 当前页面
- 最近操作
- 当前 focus 状态
- 用户是否刚结束专注

### User Memory
未来可结合：
- 用户最近完成率
- 容易拖延的任务类型
- 常见中断时间段

从而让建议更个性化。

---

## 12. 与 RAG 的关系

当前 MVP 阶段可以不接 RAG。

但未来非常适合接：

### 任务拆解与专注策略知识库
例如：
- 对低能量用户，优先输出更轻的一步
- 对进行中的任务，优先输出“续做”型建议
- 对即将截止任务，优先输出“最关键的一步”

### 用户历史行为记录
例如：
- 用户过去总在晚上中断 → 不建议复杂动作
- 用户常放弃长任务 → 建议更小步开始

因此，`suggest_next_action` 是一个非常适合后续做“上下文增强”和“个性化推荐”的 Skill。

## 15. 面试表达建议

可以这样描述：

> 在完成第一个 Skill `split_task` 后，我设计了第二个 Skill：`suggest_next_action`。  
> 它不是再做任务拆解，而是在用户停滞、犹豫或不知道先做什么时，基于当前未完成任务给出一句最适合当下开始的建议。  
> 这个 Skill 的意义在于，它开始体现 Agent 系统对“当前状态”的理解，也为后续接入 Context Engine 和 User Memory 提供了自然接口。

---

## 16. 总结

`suggest_next_action` 是 FocusPet 中非常关键的第二个 Skill。

如果说 `split_task` 解决的是“怎么拆开目标”，那么 `suggest_next_action` 解决的是：

> **现在这一步，该怎么开始。**

它非常适合成为 FocusPet 从“单点 AI 能力”升级到“多 Skill Agent 能力系统”的下一步。
